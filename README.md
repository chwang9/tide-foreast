# 潮汐预报爬虫

一个用于从 https://www.tide-forecast.com/ 抓取白天低潮数据的 Python 脚本，帮助规划潮汐池之旅！

# 从源代码安装

1. 一些设置：
   ``` 
   cd /tide-forecast-scraper`
   pip install -r requirements.txt`
   chmod +x tide-scraper.py
   ```

# 运行应用程序

爬虫只需要一个包含有效位置列表的文件名。项目中包含一个示例文件 `basic-input.txt`。尝试创建你自己的！

```
./tide-scraper.py basic-input.txt
```

# 方法论

简而言之，抓取这些数据的过程包括：请求给定位置的页面，在其中定位潮汐表，然后顺序读取表的行，管理白天状态并记录低潮以生成最终输出。

## LXML

作为网络爬虫的新手，这个库使我能够以直接的方式遍历 HTML 响应，同时获得成功解析数据所需的必要理解。

## 请求位置的潮汐表页面

理解页面如何提供所需数据很重要，因为每种表示方式都可能带来不同的注意事项。给定位置的 "tides/latest" 页面显示了所需数据，相比其他页面有 3 个优势：
  1. 可预测的解析方式
  2. 包含多天的数据
  3. 可以使用有限状态机轻松跟踪白天低潮

潮汐表页面有一个注意事项：某些位置将包含州/省名称，而其他位置仅包含位置名称。例如，Huntington Beach, California 简单地表示为 `../Huntington-Beach/...`，但 Providence, Rhode Island 表示为 `../Providence-Rhode-Island/..`。我为此添加了一个特殊情况，但如果失败，该位置将被跳过。

## 解析表格

潮汐表包含一组事件：
 * 日出
 * 月出
 * 日落
 * 低潮
 * 高潮

为了读取所有白天低潮，我遍历每一行，确定它代表什么事件。遍历行只是对表中 `td` 内的所有 `tr` 进行 xpath 操作：
 * 日出：设置 `daylight` = `True`
 * 日落：设置 `daylight` = `False`
 * 当 `daylight` 为 `False` 时，忽略当前 `tr` 迭代
 * 低潮：（加上 `daylight = True`）：记录低潮的时间和高度（以英尺为单位）
 * 高潮、月出被忽略

通过监控何时是白天，不必从完整的低潮集合中解析白天低潮。确定给定区域的日落可能很棘手，但表格为我们处理了时间顺序。
